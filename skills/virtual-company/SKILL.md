## 🎯 使用方式

### 调用团队成员

```
# 搞钱特战队
"让市场猎手分析这个赚钱机会"
"叫商业顾问评估可行性"

# 软件开发团队
"让程序员写一段代码"
"叫架构师设计系统架构"

# 小说+漫剧团队
"让小说专家写小说开头"
"叫AI提示词工程师设计提示词"
```

### 智能模型切换

```
"代码模式" → qwen3.5-plus
"深度推理模式" → qwen3-max
"日常模式" → glm-5
"图片/视频/音频" → qwen3.5-plus（多模态）
```

## 🔄 模型分配原则

| 模型 | 适用角色 | 原因 |
|------|---------|------|
| **glm-5** | 一般任务成员 | 日常分析、运营、设计等 |
| **qwen3.5-plus** | 创作/技术成员 | 写代码、写小说、创意等 |
| **qwen3-max** | 深度思考成员 | 架构、战略、法律等 |

## 🛠️ 管理脚本

### 初始化办公室
```powershell
pwsh scripts/init-offices.ps1
```

### 检查办公室状态
```powershell
pwsh scripts/check-offices.ps1
```

### 自动修复
```powershell
pwsh scripts/check-offices.ps1 -Repair
```

## 💡 核心特性

| 特性 | 说明 |
|------|------|
| 🏢 **固定办公室** | 每个成员有独立的持久session |
| 🧠 **持久记忆** | 记住所有对话，不会失忆 |
| 🎯 **专属模型** | 按角色分配最优模型 |
| 🔄 **智能切换** | 自动识别任务类型切换模型 |
| 📋 **一键管理** | 脚本自动化管理 |

1. **首次使用**：运行 `init-offices.ps1` 初始化所有办公室

3. **模型固定**：创建session时指定模型，之后不再切换
